cocos2d-html 学习笔记 1: 基本结构

cocos2d-html 学习笔记第一期

根据官方演示 hello world 来了解一些基本的操作

最近对这个游戏引擎蛮感兴趣的. 抛开webGL的部分. 从头开始看看2d部分是怎么工作的吧.

官方范例1: hello world

这个范例在 ./HelloHTML5World 目录中

通过这个范例主要介绍了cocos2d-html的启动.

目录结构

index.html里加载了cocos2d.js, cocos2d.js里定义了document.ccConfig用作cocos2d的环境变量.

box2d和chipmunk是两个物理引擎.

engineDir 和 SingleEngineFile 是加载cocos2d的两种方式. engineDir感觉是开发者模式么,会调用引擎中的jsloader.js加载引擎目录下的各种文件. SingleEngineFile听起来是把引擎编译成一个文件加载的模式.

appFiles会帮忙加载数组里的js文件.这里加载了src目录下的两个文件.

通过动态加载jsloader.js,就把cocos2d引进页面上了.

cocos2d会自动加载工程根目录下的main.js文件,这是cocos2d的入口.

这里创建了一个继承cc.Application的类,并生成了它的实例

HelloWorldScene 是在myApp.js下定义的一个cc.Scene实例.

resource.js申明一个数组去定义一些资源文件的路径.

src/myApp.js就是用户自己的脚本了.创建了一些Scene和Layer的实例.

cocos2d的结构和我接触的其他引擎差不多

Application -> Director -> Scene -> Layer -> Sprite 的结构

Application 用来管理整个游戏的进程

Director 用来管理显示的Scence以及一些全局效果, 当然了, Director本身是单例

Scene 就是画面上场景的管理. 一个时间只能有一个 Scene 的实例处于活动状态

Layer 是 Scene 上一组 Sprite 的集合. 例如游戏中可能把操作区分为一个 layer, 状态栏是另外一个, 便于统一的管理

Sprite 用来显示图像.

最后关于cocos2d的类与实例的实现. 主要是继承的模式来进行的.

首先创建一个容器来注入自己的方法, 再生成容器的实例.
