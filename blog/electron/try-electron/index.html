electron 采坑记

最近在用 electron 打包一个 webpack 生成的项目, 感觉自己踩了1万个坑. 当然主要原因还是我个人对 electron 只停留于了解的程度, 对于各种实践也是一边摸索一边进行.

绝对路径转换

webpack 的项目构建的包引用资源使用的是绝对路径, 如果 electron 加载的 html 方式是

那么 index.html 中绝对路径引用的资源, 比如 <img src="/a.png" /> 就会被定位到 file://a/png 上.

当然只要把 webpack 的 publicPath 改为相对引用即可. 不过本着”自扫门前雪”的原则, electron 的问题还是想在 electron 中解决.

对比这个场景里 http:// 和 file:// 加载的不同, 最大区别是 http 协议后面有一个域名来标记绝对路径的范围, 而 file 协议后面直接就跟上了路径, 在其范围内, 绝对路径是指向根部的.

所以我们只需要想办法给 file 协议也加上一个域, 再在 electron 里处理就好了.

electron 提供的 protocol 接口就可以用来注册或拦截一个协议.

使用 file 协议比起自定义协议有两点好处.

一是不用额外处理 CORS 问题. 由于和服务端的通信还是走 htts? 协议, 对于 local:// 的环境下, 请求需要响应头提供一部分跨域头. 当然在 electron 中也可以通过 webRequest手动的拦截 http 响应并注入 headers

(BTW, electron header 头的内容必须是一个数组, 这里也稍微坑到我了一下)

另外一个区别则是目标版本的 electron, 自定义协议用不了 service work.
